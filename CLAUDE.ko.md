# 수학의 기초와 기본 (Math Basic) 가이드라인

이 문서는 **Math Basic** 프로젝트를 위한 핵심 기술 스택, 작업 원칙 및 AI 작업 권한(Permissions)을 정의합니다.

## 1. AI 작업 프로세스 (필수 준수)

모든 작업은 반드시 다음 4단계 프로세스를 따르며, 단계별로 사용자와 소통합니다.

1.  **이해 (Understanding)**: 사용자의 요청을 분석하고, 목표와 제약 조건을 정의합니다. 모호한 점은 질문을 통해 명확히 합니다.
2.  **계획 (Planning)**: 구체적인 수정 파일 목록, 변경 로직, UI/UX 디자인 등을 제안합니다. **사용자의 명시적 승인**을 받은 후 다음 단계로 넘어갑니다.
3.  **실행 (Execution)**: 승인된 계획에 따라 코드를 작성하고, 테스트 및 검증을 수행합니다.
4.  **반성 (Review)**: 작업 결과를 요약하고, 발생한 문제나 향후 개선점, 사용자 확인이 필요한 사항을 공유합니다.

## 2. 워크플로우 및 구조

- **Raw (`/raw`)**: 수집된 원본 및 기초 자료.
- **Vault (`/vault`)**: 정리된 지식 (Obsidian 기반).
- **Web (`/web`)**: 디지털 출판 (Astro 기반).
- **Scripts (`/scripts`)**: 자동화 및 검증 도구.

## 3. 핵심 기술 스택

- **프레임워크**: Astro
- **수학 엔진**: KaTeX (로컬 라이브러리 사용: `/public/lib/katex`)
- **인터랙티브**: JSXGraph (로컬 라이브러리 사용: `/public/lib/jsxgraph`)
- **디자인**: (신규 UI 작업 예정 - Headless 스타일 지향)

## 4. [Permissions] AI 권한 및 작업 제한

AI(Antigravity)는 프로젝트의 안정성을 위해 다음 규칙을 준수해야 합니다.

1.  **계획 승인 필수**: 어떠한 코드 수정이나 시스템 변경도 **'계획(Planning)' 단계의 승인 없이 실행할 수 없습니다.**
2.  **제한 구역**: `/web/public/lib/` 하위의 라이브러리 파일은 직접 수정할 수 없습니다. (업데이트 필요 시 사용자 승인 필수)
3.  **수정 금지**: `/raw` 폴더의 원본 파일은 읽기 전용으로 사용하며, 가공된 결과물만 `/vault`나 `/web`에 작성합니다.
4.  **명령 승인**: `rm` (삭제), `npm install` (패키지 추가) 등 시스템 변화를 일으키는 명령은 반드시 사용자 승인을 거칩니다.

## 4. 수학 수식 작성 규칙 (필수)

Astro 컴포넌트 내에서 안정적인 렌더링을 위해 다음 규칙을 엄격히 적용합니다.

1. **캡슐화**: 모든 수식은 `{ "$ ... $" }` 로 감싸야 합니다.
2. **이중 백슬래시**: LaTeX 명령어는 반드시 이중 백슬래시(`\\`)를 사용합니다. (예: `\\frac`, `\\to`, `\\sqrt`)
3. **검증**: 모든 문서 작성 후 `scripts/check-formula.js`를 실행하여 수식 오류를 확인합니다.

## 5. UI/UX 디자인 원칙 (신규)

- **가독성 최우선**: 기초 수학 학습자를 위해 폰트 크기와 줄 간격을 넉넉하게 유지합니다.
- **시각화 강조**: 개념 설명 시 가능한 한 JSXGraph를 활용한 인터랙티브 요소를 포함합니다.
- **미니멀리즘**: 불필요한 장식을 배제하고 핵심 개념이 돋보이는 디자인을 지향합니다.

## 6. 안정성 및 최적화 (Advanced)

1. **에셋 관리**: 모든 이미지는 최적화된 포맷(WebP 권장)을 사용하고 `/public/assets` 내에 챕터별로 구조화하여 저장합니다.
2. **데이터 구조화**: 각 페이지 상단(Frontmatter)에 `difficulty`, `topic`, `keywords`를 명시하여 향후 검색 및 인덱싱에 대비합니다.
3. **폴백(Fallback) UI**: JSXGraph 등 동적 라이브러리 로드 실패 시에도 사용자가 학습 내용을 이해할 수 있도록 대체 텍스트나 정적 이미지를 제공합니다.
4. **중복 최소화**: 자주 사용하는 수학적 정의나 상수는 `src/utils/` 하위에서 공통 모듈로 관리합니다.

## 7. AI 및 검색 최적화 (AEO/GEO) 규칙 (필수)

모든 페이지는 AI(Gemini, GPT 등)와 검색 엔진이 콘텐츠를 완벽하게 이해할 수 있도록 다음 규칙을 반드시 준수해야 합니다.

1.  **SSR(Server Side Rendering) 우선**:
    *   모든 수학 문제, 개념 설명 등 핵심 콘텐츠는 클라이언트 사이드 자바스크립트가 아닌, Astro의 **Frontmatter(서버 사이드)**에서 미리 렌더링되어 HTML 소스에 포함되어야 합니다.
    *   수식은 `katex.renderToString`을 사용하여 서버에서 HTML로 변환하여 주입합니다.
2.  **구조화 데이터(JSON-LD) 삽입**:
    *   각 페이지의 성격(Quiz, LearningResource 등)에 맞는 `application/ld+json` 스크립트를 포함하여 AI가 페이지의 의도를 즉시 파악하게 합니다.
3.  **디자인 무결성 유지**:
    *   최적화 작업 시 기존의 CSS 레이아웃과 시각적 디자인이 변경되지 않도록 주의합니다. (`set:html` 활용)
